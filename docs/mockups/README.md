# UI Mockups — 给人看,也给 AI 看

本目录是 [04 设计系统](../04-design-system.md) 的**可视化配套物**,用途:在用任何 AI 模型(GPT、Claude、Gemini…)辅助实现前端时,把这里的文件直接喂给它,避免模型凭空想象出不符合设计的 UI。

| 文件 | 是什么 | 怎么用 |
|---|---|---|
| `ui-mono.html` | **全界面定稿 mockup**(纯墨方案 × 亮/暗双主题),覆盖软件所有界面:HUD 全部状态(听写/翻译/助手/原样/无声/处理中/成功/失败/错误变体)、托盘菜单(就绪/录音中)、助手面板(有选区/初始态)、设置窗口全部 9 页 + ProviderCard 编辑态、首次启动引导 5 步、主页(统计卡片 + 历史记录,含空状态)、组件库全态、图标与 logotype、色板 | 浏览器打开对照着实现;喂给 AI 时直接把整个文件贴进上下文——CSS 里的色值、圆角、间距就是规格本身。**实现某个界面时,把该界面对应的 HTML 片段(或整个文件)连同下方约束一起给模型** |
| `tokens.css` | 可直接引用的 design tokens 起步文件(色彩 + 字体 + 圆角 + 间距,含双主题切换逻辑) | 建前端工程时复制为 `src/styles/tokens.css`,之后以工程内文件为准 |

## 给 AI 的关键约束(喂 mockup 时建议一并粘贴)

1. 全部颜色引用 `tokens.css` 变量,**禁止硬编码色值**。
2. 界面只有黑白灰;唯一彩色是 `--recording` / `--error`(录音红/错误红),不得引入任何其他色相与渐变。
3. 选中态 = `--sel-bg` 灰底 + 600 字重(+ ✓),**禁止反色实底和粗黑描边**;每个页面只有主按钮一处实底反色。
4. 链接与 Ghost 按钮用下划线表达,不靠颜色。
5. 图标/logotype 只用纯黑纯白:浅色模式黑底白 glyph,深色模式白底黑 glyph;`Typex` 五字母永远同色。

> 冲突时的优先级:[04 设计系统](../04-design-system.md) 的 token 表 > `tokens.css` > mockup HTML。改设计先改 04 章,再同步这里。界面的行为语义(状态机、快捷键、错误文案全表)以 [05 UX 规格](../05-ux-spec.md) 为准,mockup 只定外观。
