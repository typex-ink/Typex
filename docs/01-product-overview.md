# 01 · 产品概述

> Typex 产品设计书 · 第一章
> 状态：草案 v0.1（2026-07-03）｜关联章节：[02 功能规格](02-features.md) · [09 决策清单](09-decisions.md)

---

## 1. 一句话定位

**Typex 是一款开源、跨平台（Windows / macOS / Linux）的 AI 语音输入软件：按住一个键说话，松开后被整理干净的文字出现在任何应用的光标处——并且能翻译你说的话、能听懂你对选中文本的指令、能随时语音提问得到回答。**

口号（候选）：

- 中文：**「说，即所得。」**
- 英文：**"Speak. It types."**

## 2. 为什么做 Typex（与 Typeless 的关系）

Typex 对标 Typeless（2025–2026 年最流行的 AI 语音输入软件之一），但在三个根本点上做出不同选择：

| 维度 | Typeless | Typex |
|---|---|---|
| 源码 | 闭源 | **开源（GPLv3**，[ADR-9](09-decisions.md)） |
| 模型 | 绑定官方云端，必须注册账号 | **BYOK（Bring Your Own Key）**：用户自选 STT / 翻译 / Agent 模型的端点与密钥，无账号体系 |
| 价格 | $12–30/月 | **免费**（用户只为自己的 API 用量付费） |
| 平台 | Mac / Win / iOS / Android | **Mac / Win / Linux**（桌面三平台并重，Linux 是同类产品普遍缺失的） |
| 隐私 | 音频经官方服务器，曾有「营销本地化 vs 实际云端」信任争议 | 音频只发往**用户自己配置的端点**，无中间服务器，代码可审计 |

### 2.1 竞品调研摘要（2026-07）

- **Typeless**：核心价值是「重写而非转写」——去语气词、自动标点、口述中改口识别（说"不对，改成……"只保留最终意图）、选中文字语音改写、100+ 语言实时翻译。头号抱怨：6 分钟会话硬上限、高峰期云端报错且不重试（丢内容）、$30/月月付最贵、权限索取过宽、隐私营销与云架构落差。
- **Wispr Flow**：品类老大（融资 $81M），AI 重写最激进，但曾因定期截屏取上下文引发隐私风暴；$15/月。
- **superwhisper**：全本地 Whisper 推理、$249.99 买断，隐私党首选；AI 重写与易用性弱于云端派。
- **Aqua Voice**：自研模型主打技术词汇/代码精度、流式上屏、约 $8/月。
- **开源侧**：Handy（Tauri 2 + Rust，本地转写，无 AI 能力）、Whispering（Tauri 2 + Svelte，BYOK 转写，无注入外的 AI 工作流）——**目前没有一个开源项目同时做到「体验对齐 Typeless + BYOK + 三平台」，这就是 Typex 的位置。**

### 2.2 从竞品得到的三条设计铁律

1. **绝不丢用户说的话。** 云端失败必须自动重试；任何一次录音的原始转写稿必须可找回（可选历史记录，见 [02 功能规格 F-7](02-features.md)）；**没有会话时长硬上限**（长录音自动分段续接，用户无感）。
2. **权限最小化，且声明与实现严格一致。** 只申请麦克风 + 辅助功能（注入/读选区必需），不碰屏幕录制、摄像头、浏览历史。开源本身是信任背书，要在 README 和官网大声说明数据流向。
3. **快捷键不抢占用户已有习惯。** 默认键位避开系统与常用软件占用，首次启动即可改（见 [05 UX 规格 §7](05-ux-spec.md)）。

### 2.3 商业模式（[ADR-16](09-decisions.md)）

软件本体永久开源免费、不内置收费功能。盈利点是后续自营的 **Typex 官方 STT 套餐**：面向不想自己研究 API 的用户，提供开箱即用的官方 Provider 预设。边界：官方服务与第三方预设同列无特权、必须走标准 OpenAI 兼容协议、客户端零上报——BYOK 与可审计性是产品的信任根基，不因商业化打折。

## 3. 目标用户

按优先级：

1. **双语/多语工作者**（中文用户为先发人群）：日常在中英之间切换，语音翻译是刚需——说中文、出英文，直接进 Slack / 邮件 / 文档。
2. **重度文字工作者与开发者**：写文档、回消息、写 issue，追求「说话比打字快」；开发者同时是开源软件的天然传播者和贡献者。
3. **隐私敏感与自托管人群**：不接受音频经过第三方中间服务器，愿意自配 API（含本地 / 自建端点）。
4. **Linux 用户**：同类商业产品全部缺席的市场。

非目标（v1 不做）：移动端、会议转录/字幕、团队协作、音频文件批量转写。

## 4. 产品原则

1. **快是第一体验。** 从松开按键到文字上屏，短句目标 < 1.5 s（云端）；所有 UI 反馈 < 100 ms。性能预算见 [07 代码架构 §12](07-code-architecture.md)。
2. **安静的工具。** Typex 是输入法级的常驻软件：无常驻主窗口（主页按需从托盘打开）、活在托盘里、空闲时接近零占用；不弹通知、不求关注。
3. **默认好用，一切可配。** 开箱即用的默认值（预设 Provider 模板、默认快捷键、自动语言检测），但每一层都允许高级用户替换——模型、端点、提示词、快捷键、注入方式。
4. **优雅但不花哨。** 视觉语言克制、留白充分、动效服务于状态传达（见 [04 设计系统](04-design-system.md)）；尊重系统深浅色模式与「减弱动态效果」设置。
5. **失败也体面。** 每个错误都有明确原因与下一步动作（重试 / 换 Provider / 查看权限指引），绝不静默失败。

## 5. 核心场景（v1 三大功能的用户故事）

**S1 · 语音输入**：小雨在微信输入框按住右 Ctrl，说「嗯……今晚的会改到八点、呃不对、八点半，你们先吃饭不用等我」，松开——1 秒多后输入框里出现的是整理干净的「今晚的会改到八点半，你们先吃饭不用等我」：语气词没了、改口只留最终意图、标点正确。这背后是 STT + 轻量文本整理模型两级流水线。

**S2 · 语音翻译**：小雨切到 Slack 要回复美国同事，按住翻译键说中文「这个 bug 我明天上午修，今天先给你个临时绕过的办法」，松开——英文 "I'll fix this bug tomorrow morning; for today, here's a temporary workaround." 直接上屏。

**S3a · 选中改写**：小雨选中自己刚写的一段生硬的英文邮件，按下助手键说「语气礼貌一点，再精简 30%」——选中文本被原地替换为改写结果。

**S3b · 语音问答**：小雨在浏览器里选中一段报错日志，呼出 Typex 助手问「这个错误一般是什么原因？」——助手面板流式给出回答，复制或插入到光标处即可。什么都不选也能随口问一句、立刻得到答案（单轮问答，刻意不做成完整 Agent——重度 Agent 工作流交给 Codex / Claude Code 这类专业工具）。

## 6. 名字与品牌注记

- **Typex** = **Type** × **X**（X：变换/翻译/智能——「说话经过一次变换成为文字」）。
- ⚠️ **命名风险备忘**：Tipp-Ex 是 BIC 旗下修正液品牌（欧洲知名度高）；Typex 也是二战英国密码机名。软件类别与两者均不冲突，开源项目风险低，但**若未来做商业分发/商标注册需重新评估**（记录于 [09 决策清单 D-13](09-decisions.md)）。
- 官方域名：**typex.ink**（已注册，[ADR-12](09-decisions.md)）。
- 品牌视觉（图标、配色、字体）详见 [04 设计系统](04-design-system.md)。
