## 2. 常驻与托盘

- 启动后无任何窗口，仅托盘图标（状态图标规格见 [04 §2.4](04-design-system.md)）。
- 托盘菜单结构：

```
● Typex 就绪                    ← 状态行（不可点，随状态变化：就绪/录音中/处理中/已暂停/错误摘要）
─────────────────
复制上次结果
─────────────────
文本整理           ✓ 开
翻译目标            EN ▸       ← 子菜单：最近语言 + 更多…
模型                  ▸        ← 子菜单：听写/翻译/问答 三组已存档案，✓ 标当前，点选即切（F-4 多档案）
暂停 Typex
─────────────────
设置…               主页…
检查更新             退出
```

- 左键点击托盘 = 打开主页；右键 = 菜单（macOS 均为菜单，符合平台惯例）。
- 选择「暂停 Typex」立即禁用全局触发，并静默取消当前录音（含短按 toggle）；已进入转写/处理的会话继续完成。恢复后只接受新的完整按键手势，不沿用暂停前的 held 状态。
- Windows 低级键盘钩子在运行期失效时，立即静默取消当前录音并把托盘状态改为 `⚠ 快捷键不可用`；同一终态不得重复取消或反复提示。应用主动退出导致的正常 hook shutdown 不显示错误。

## 3. HUD（核心界面）

### 3.1 形态

屏幕**底部居中、距底边 48px** 的全圆角胶囊，高 44px，宽度随状态 160→320px 平滑过渡；胶囊本体始终使用完全不透明的 `--surface`，不使用透明混合或毛玻璃，仅胶囊外的原生窗口区域透明；**永不获得焦点**。多显示器时出现在「当前焦点窗口所在屏」。胶囊内容导致原生透明窗口尺寸变化时，必须以一次原生 frame 更新同时完成缩放与重新居中，禁止分两次更新尺寸和位置而暴露中间偏移帧。

### 3.2 状态与布局

**录音中（听写）**：

```
╭──────────────────────────────────╮
│ ● 0:07   ▁▃▅▇▅▃▁▂▄▆▄▂▁   [听写] ✕│
╰──────────────────────────────────╯
   │        │                │    └ 取消（Esc）
   │        │                └ 模式徽标（听写/中→EN/助手）
   │        └ 实时波形（--voice 色，Canvas）
   └ 录音点（--recording，1s 呼吸）+ 计时
```

- 翻译模式：徽标显示 `中 → EN`（点击弹出目标语言快切）；助手模式：徽标 `助手`。
- 波形必须真实反映电平——用户要能一眼确认「麦克风在收音」。收不到声音 3 秒 → 波形区显示「没有听到声音」微提示。

**处理中**：松开快捷键后先切换为处理态，再异步完成音频收尾；目标为 100 ms 内显示五柱依次呼吸动画 + 文案（`正在转写…` → `正在整理…`/`正在翻译…`）。呼吸动画按真实经过时间保持 1.6 s 固定周期，不随平台或显示器刷新率变化；超过 5 s 显示已耗时。

**成功**：胶囊短促回弹，显示 `✓ 已输入`（`--success`），600 ms 后下滑淡出。

**失败**：

```
╭──────────────────────────────────────────╮
│ ⚠ 转写失败：无法连接 API    [重试] [复制原文] ✕ │
╰──────────────────────────────────────────╯
```

- 错误文案 = 错误分类的人话（见 §9）；`复制原文` 仅在 STT 已成功、后续步骤失败时出现（不丢话铁律）。失败态不自动消失。

**原样模式提示**：整理层关闭时，录音态徽标处加小字 `原样`。

### 3.3 状态机与时序

```
hidden ─按键按下─▶ recording ─松开─▶ processing ─▶ success(600ms) ─▶ hidden
   ▲                  │ Esc/✕            │ 失败                        
   └──────────────────┴◀────────────── failed ─重试─▶ processing
                                          └─✕/超时120s─▶ hidden
```

出现动画 220 ms spring 上滑；按下到 HUD 可见 ≤ 100 ms（性能预算）。

**重入规则**：Transcribing/Processing/Injecting 期间再次按下触发键 → 本次按键忽略，HUD 轻微晃动 + 微文案「正在处理上一条…」（当前不做流水线并行，简单可预期；失败态下按触发键 = 放弃上一条并开始新录音）。

**Esc 取消的边界**：启用「Esc 取消当前流程」时，Recording、Transcribing（含音频收尾）、Processing、尚未提交 OS 输入的 Injecting 与 Failed 都可取消；取消终止当前 STT/LLM 任务、关闭生成中的回答窗，并使迟到回调失效。Injecting 以首个 OS 粘贴/键盘输入调用为原子提交边界：Esc 先取得取消权则恢复剪贴板且不发送输入事件，注入先提交则本次 Esc 不再取消并让流程正常完成。只有成功认领当前会话取消权的那一次物理 Esc 才被消费：吞掉初次 down、自动重复与配对 up，并只发送一次带 `session_id` 的取消；空闲、设置关闭、过期会话或注入已经提交时 down/up 均完整透传。取消后退出动画期间的新 Esc 也不拦截，其他键盘输入始终放行。回答弹窗自身的 Esc/关闭/失焦不受此设置影响。

## 4. 回答弹窗（F-3）

> 只读展示窗：**仅当有回答要展示时出现**（对选区提问 / 无选区提问，[02 F-3](02-features.md) / ADR-23）。改写型指令直接替换选区，不经过本窗；语音输入全部走全局助手键，弹窗自身没有任何输入能力。

### 4.1 形态

优先显示在选中文本下方（放不下时上方），选区位置不可得或无选区时回退到屏幕上 1/3 居中；宽 560px，高度随内容收缩/展开（透明区域不得拦截点击），大小不可手动调整，圆角 16。面板从开始、流式生成到完成/错误始终使用不透明 `--surface` + 边框，不使用 `color-mix` 透明混合或毛玻璃；只有圆角外的原生窗口区域透明。禁用系统原生窗口阴影与外扩 CSS 阴影；显示时获得焦点；**失焦（焦点切到其他应用）自动关闭**，无固定（📌）选项。Windows 使用前台窗口所在显示器的 work area 和每显示器 DPI 计算位置，必须支持负坐标副屏。

### 4.2 布局

```
╭────────────────────────────────────────────────╮
│ 🎤 总结一下这段文本，告诉我要点。            ✕  │  ← 指令回显行 + 关闭按钮
│    ⌗ 选中内容 · 217 字                          │  ← 选区摘要（有选区时）
├────────────────────────────────────────────────┤
│  （流式回答区域，Markdown 渲染，文本可选中复制） │
│                                                │
│  Typex 是一款开源、跨平台的 AI 语音输入软件…    │
│                                                │
╰────────────────────────────────────────────────╯
```

### 4.3 行为

- **单轮语义**：每次提问独立（不携带上一问）；新提问重置弹窗内容；无会话列表、无历史线程、无动作按钮。
- **上下文读取**：松开助手键（录音结束、转写开始）时读取当前选中文本——不能在按住期间读，剪贴板降级的模拟 Cmd+C 会触发组合键让路误取消会话（[06 §7.6](06-code-architecture.md)）；macOS 剪贴板降级使用固定 CGEvent 键码，不经输入法布局查询。Windows 优先使用 UIA 的明确空/非空结果；仅在 Ctrl+C 降级时读取已知编辑器的空选区元数据，VS Code/Monaco 的 `vscode-editor-data` 必须是 `version=1` 且 `isFromEmptySelection=true` 才按无选区处理。缺失、损坏或未知元数据不得用文本长度/换行启发式丢弃内容；没有可靠 marker 的第三方编辑器若把无选区 Ctrl+C 定义为复制整行，仍可能被当作选区，这是明确的兼容性残余。
- **弹出时机**：无选区 → 转写完成、LLM 请求发出即弹窗并流式渲染；有选区 → 流首部确认为回答型（`ANSWER:` 信号，见 [03 §3.4](03-model-providers.md)）的那一刻才弹窗——改写型结果全程不弹窗（直接替换选区，反馈走 HUD）。
- **关闭**：右上角 ✕ / `Esc` / 焦点切换到其他应用。流式生成中关闭会先发送会话取消再隐藏，部分回答不写历史；完成或终态错误后关闭只隐藏。
- 流式渲染节流 30 fps；渲染 Markdown（标题/列表/代码块），链接点击用系统浏览器打开；回答文本可直接选中复制（`⌘/Ctrl+C`）。

## 5. 设置窗口

左侧图标+文字导航（宽 160px），右侧内容区。导航：`通用 · 听写 · 翻译 · 助手 · 模型服务 · 快捷键 · 历史 · 诊断 · 调试 · 关于`。

### 5.1 「模型服务」页（最重要的一页）

```
┌─────────────┬──────────────────────────────────────────┐
│ 通用         │  模型服务                                 │
│ 听写         │                                          │
│ 翻译         │  功能分配                                  │
│             │  ── 语音转文字 ────────────────────────   │
│ 助手         │  ┌ ProviderCard ─────────────────────┐   │
│ ▶模型服务    │  │ [logo] Groq · whisper-v3-turbo    │   │
│ 快捷键       │  │ openai_compat · 已连接 · 320ms     │   │
│ 历史         │  │            [测试] [编辑] [切换 ▾] │   │
│ 诊断         │  └───────────────────────────────────┘   │
│ 关于         │                                          │
│             │  ── 文本整理 ────────────────────────     │
│             │  ── 翻译模型 ────────────────────────     │
│             │  ── 问答模型 ────────────────────────     │
│             │                                          │
│             │  配置池                         [新建…]  │
│             │  STT 服务 / LLM 服务配置列表              │
└─────────────┴──────────────────────────────────────────┘
```

- 页面分两段：上方「功能分配」每个功能槽位一张 `ProviderCard`，只负责显示/切换当前服务；下方「配置池」列出全部 STT / LLM 服务配置，负责新建、编辑、测试、删除。配置池放在页面底部，避免遮挡最常用的功能分配。
- 「编辑」进入**编辑子页**（同一内容区内切换，顶部「← 模型服务」返回链接；保存/删除档案后自动返回列表；有未保存修改时离开需确认）：预设下拉（OpenAI/Groq/SiliconFlow/火山·豆包/DeepSeek/OpenRouter/Ollama/自定义，仅作表单填充、无推荐标注）→ 按 `capability/kind` 动态渲染字段（STT：openai_compat Base URL/模型/密钥 或 volcengine AppKey/AccessToken/ResourceId；LLM：Chat Completions / Responses；所有 STT / LLM 档案均显示「调用超时（秒）」数值输入，默认 60 秒；LLM 档案另显示「思考等级」下拉，默认“关闭”，选项为关闭/最小/低/中/高/极高；本地与 Qwen 兼容端点按开关语义处理）。「已下载模型」管理页同为子页，同样以「← 模型服务」返回。
- **「切换 ▾」= 从服务池选择兼容配置（[ADR-21](08-decisions.md)）**：下拉列出该功能可用的服务配置（✓ 标当前）+「新建配置…」；新建不覆盖旧档案，切换即时生效。同一组配置也出现在托盘「模型 ▸」子菜单中（快速切换，不用打开设置）。
- 新建配置在同一次编辑流程内执行「测试连接」和「保存」时必须复用同一个档案 ID；测试后保存只更新该档案并完成槽位分配，不得向配置池追加内容相同的副本。
- 密钥输入用 `SecretInput`（默认掩码、可见性切换、粘贴按钮）；保存后随 profile 写入 `settings.json` 的 `credentials` 字段，诊断包与日志必须脱敏。
- 「测试」按钮内联显示结果：成功显示 `✓ 320ms` 延迟徽标；失败时测试按钮自身变为红色「测试失败」，完整分类错误与上游报错只在悬停/键盘聚焦时显示，避免长错误文案挤压卡片布局。
- **本地 Provider 卡片**：预设列表含「本地 · 离线」；卡片副标题显示引擎与模型（如 `local · Qwen3.5-2B`）及模型状态（`已下载 · 1.3 GB` / `未下载 [下载]`，下载中显示进度条）；无密钥字段，编辑态字段 = 模型（下拉，来自模型库）+ 加载策略（常驻 / 录音时预热）；「测试」= 本地跑一次内置样音/ping。问答槽允许手动选择本地 LLM，但不参与零配置自动兜底（[ADR-20](08-decisions.md)）。
- **已下载模型管理**（页面底部「管理…」进入）：已下载列表（名称 · 用途 · 体积 · 来源 · 许可证 · 被哪些槽使用 · 删除，删除使用中模型先警告）+「导入模型」入口（选择本地 GGUF/ONNX/tokens 文件，托管导入到 Typex 模型目录）+ 模型库可下载列表（行内显示推荐硬件与本机检测结果，如「建议 GPU 加速（本机 ✓ Metal/Vulkan）」；未满足时显示「低于建议」，[ADR-22](08-decisions.md)/[ADR-26](08-decisions.md)）+ 底部占用合计与下载源切换（自动 / HuggingFace / ModelScope）。只要条目有远程下载源，低于建议仍可直接下载且不弹额外确认框；无下载源时按钮禁用。下载中进度条旁显示已下载 / 总量。
- 共用连接不再需要单独开关：多个 LLM 功能选择同一个池中服务配置，即表示共用连接。

### 5.2 其他页要点

- **通用**：语言（跟随系统/中/英）、主题（跟随系统/亮/暗）、开机自启、提示音开关与音量（0–100%，默认 60%；关闭时滑杆禁用但保留上次音量）、网络代理（跟随系统/手动/直连）、更新通道（稳定版 = 正式 release；开发版/Nightly = 最新 nightly build）。更新通道的首次默认值必须与安装包构建类型一致：带 SemVer prerelease 后缀的测试构建默认「开发版」，纯 SemVer 正式构建默认「稳定版」；用户改选后保留其选择。
- **听写**：文本整理开关、整理系统提示词（折叠；副描述只说明其用于控制听写转录的清理方式，不展示 XML、消息角色等实现细节；展开为纯文本编辑器，不包含动态占位符；「恢复默认」回到内置 system prompt；运行时消息规格见 [03 §3.4](03-model-providers.md)）、VAD 模式分段控件（「神经网络 / 能量门限」，全键盘可操作）及当前模式唯一可见的门限滑杆（神经网络 `0.10–0.90`、步长 `0.05`、默认 `0.50`；能量 `0.001–0.050`、步长 `0.001`、默认 `0.010`）。模式行须说明神经网络适合多数环境、能量门限仅按输入音量判断；门限行须说明神经模式调节的是语音置信度、能量模式调节的是输入音量，并明确两者均为数值越低越灵敏、数值越高越能过滤噪声，通常保持默认即可。其后依次为注入方式（自动/剪贴板粘贴/逐字输入）、粘贴延迟滑杆和麦克风设备选择。设备下拉显示用户可读名称，但保存稳定设备 ID；已固定设备缺失或旧名称无法唯一迁移时保留失效项并明确标为不可用，不得伪装成系统默认。文本整理默认开启；关闭后听写、翻译、助手都直通原始 STT 文本，不经过整理层。
- **翻译**：源语言（你说话使用的语言）、目标语言（要翻译成的语言）、双向翻译开关（检测到说的是目标语言时自动译回源语言）、翻译系统提示词（折叠，置底；副描述只说明其用于控制翻译质量、语气、格式与输出方式，不展示运行时消息格式）。
- **助手**：选区处理系统提示词与无选区问答系统提示词（均折叠；标题不附加「高级」标记；副描述分别说明选区处理/相关问答和无选区语音问答用途，不展示 XML、消息角色或内部输出协议；可单独展开，未编辑的另一项仍显示入口；不包含样例或动态占位符）。
- **历史**：开关（默认开，附一句说明：仅存本机、不含音频）、保留期限（默认 90 天）、清空按钮。
- **诊断**：环境自检（平台/compositor/可用注入后端/权限状态，每项 ✓/✗ + 修复引导按钮）、日志目录、导出诊断包（自动脱敏）。
- **Windows 诊断**：额外显示低级键盘钩子、SendInput、UI Automation、WebView2、当前进程完整性级别与本地模型 CPU/Vulkan 状态；只显示能力与错误分类，不暴露 HWND、窗口标题全文或选区内容。
- **调试**：开发/测试入口；提供「重新进入引导页」按钮，用于手动打开首次启动引导流程，不修改 `onboarding_done` 状态。
- **关于**：图标、Typex logotype、版本号、许可证、官网、隐私口号、检查更新、开源许可入口。点击「开源许可」用系统浏览器打开 GitHub 仓库 `LICENSE`；点击官网链接打开 `typex.ink`。检查更新发现新版本后只展示确认卡片，下载安装必须由用户主动点击确认。

## 7. 快捷键

### 7.1 默认键位（已定，[ADR-7](08-decisions.md)）：全修饰键三角方案

| 功能 | macOS | Windows / Linux | 行为 |
|---|---|---|---|
| 听写 | 按住 `右 ⌘` | 按住 `右 Ctrl` | 长按=说话，短按=切换录音 |
| 助手 | 按住 `右 ⌥` | 按住 `右 Alt` | 长按=说出指令/提问，短按=切换录音 |
| 翻译 | 按住 `右 ⌘ + 右 ⌥` | 按住 `右 Ctrl + 右 Alt` | 两键按住（不分先后）=翻译录音 |

设计逻辑：**只用修饰键，不碰任何普通键**。修饰键「单独按住」在几乎所有软件里是空操作，右侧修饰键又极少参与日常组合键——这是冲突概率最低的一类键位；三键有记忆结构（A=输入，B=问答，A+B=输入的变体：翻译）。

**乐观启动与模式判定**：任一已确认触发手势开始可见会话；按住期间若补全一个严格包含当前 chord 的更长 chord，则无缝切换为后者对应模式（音频保留、HUD 徽标即时变化）。默认键位中这表现为在听写或助手手势上追加另一键后切换为翻译。Windows 默认右 Ctrl / 右 Alt 单键进入最多 75 ms 的静默确认窗口时，立即以唯一 token 启动内存候选录音；出现普通键/AltGr 则静默取消并释放，出现第二触发键并补全更长 chord 时立即确认并切换，单键超时确认则将同一流提升为正式录音。候选阶段无 HUD、提示音、电平、历史、磁盘或 Provider 副作用；单键确认后的 HUD 仍须满足按下到可见 ≤ 100 ms。

**自定义 chord 语义**：三个 `HotkeyRecorder` 分别保存听写、助手与翻译的完整 chord；数组内全部键按住后才算手势开始，partial chord 不启动录音。三组 chord 必须非空；听写与助手不能相同或互为子集，翻译不能与任一前两项完全相同，界面就地显示校验提示并阻止保存。翻译可以包含前两项的按键，也可以是其中一项的严格子集；任一已启动 chord 被更长 chord 完整包含时，较长 chord 接管模式并保留音频，没有包含关系的额外完整 chord 则保持最先启动的模式。随后沿用既有 release 语义，等本次 tracked 触发键全部释放才结束。录制普通键时按物理位置保存稳定 `KeyId`，换 QWERTZ 等布局或切换输入法不会把同一位置变成另一绑定，展示标签仍按当前平台惯例本地化。

`HotkeyRecorder` 空闲时按钮显示「更改」，录制时只把同一按钮文案切换为「按下键位…」，不得在设置行下方额外展开录制提示块。`Esc` 仍静默取消且不修改原绑定；录制完成后的快捷键冲突警告继续单独显示。Windows WebView2 若把同一物理修饰键的 keydown / keyup 报成不同侧，录制器必须按修饰键家族完成配对，并保留事件中可识别的右侧身份；若录制期间确实收到 `Unidentified` / `Process` keydown，而对应 keyup 可识别，则以 keyup 恢复该键并完成 chord，不得退化为只保存其他已按下键；没有此前未知 keydown 的孤立 keyup 必须忽略。任一 keyup 仍表明其他修饰键处于按下状态时不得提前提交，须继续监听到最后一个修饰键释放；进入该等待后 2 秒仍未收到最终 keyup，或窗口失焦时，静默取消本次录入并保留原绑定。

**配套规则（实现必须遵守，见 [06 §7.3](06-code-architecture.md)）**：

1. **组合键让路**：按住触发键期间若按下任何**普通键**（用户实际在用 `⌘C`、`Ctrl+C`、`AltGr+E` 等系统/输入组合键），立即静默让路并完全放行按键。Windows 在 75 ms 确认窗口内只能存在不可见候选流，不得触发 HUD、提示音、电平或业务会话；普通组合键取消候选，确认窗口后已经启动的正式录音则立即静默取消。这条规则是单修饰键触发可行性的根基。
2. **Esc 选择性吞键**：Windows 在现有低级钩子中、macOS 在 grab event tap 中，仅吞成功认领会话的 Esc down/repeat/up；认领失败与所有非 Esc 键鼠事件原样返回。Linux 当前仍为 listen-only，待平台后端正式适配。
3. **Windows Alt 单击吞键**：单击 Alt 会聚焦应用菜单栏，助手键短按（切换录音）时须经低级键盘钩子吞掉该次 keyup，防止菜单栏被误激活。
4. AltGr 布局（欧洲键盘）：右 Alt 物理键即 AltGr——规则 1 已保证 `AltGr+字母` 打特殊字符不受影响；单独按住 AltGr 说话无副作用。
5. **漏 release 自恢复**：底层监听偶发漏掉触发键 release 时，下一次同触发键 down 不得永久静默；判定器应在短暂抖动窗口后重置旧状态并重新触发，最多丢弃上一段卡住的录音，不让快捷键进入“按了没反应”的持久状态。
6. Windows 低级钩子必须忽略 Typex 自己通过 SendInput 注入的事件；`Right Ctrl+C`、物理 `Right Alt+普通键` 与典型 AltGr 原始序列（伪 `Left Ctrl` + `Right Alt`）在确认窗口内产生普通键时，不得产生任何 HUD、提示音或录音副作用。

**已知残余情况**（onboarding 第 4 步当场改键兜底）：MacBook 内置键盘无右 Ctrl（故 macOS 用右 ⌘）；用户用 Karabiner/PowerToys 重映射过右侧修饰键；HHKB 等无右修饰键的小配列键盘。`HotkeyRecorder` 支持单修饰键、修饰键组合与普通键，录制时对系统热键表与常见软件冲突黄条警告；另提供预设建议：CapsLock（提示 macOS 中文用户注意中英切换冲突）、F13–F19、右键菜单键（仅 Win/Linux）。历史配置中的 `Return`/`Num1`/`LeftArrow`/`AltGr` 等后端名只作为迁移别名接受，新录制值统一保存为稳定 `KeyId`。

### 7.2 回答弹窗内快捷键

`Esc` 关闭；若回答仍在生成则同时取消当前助手会话。`⌘/Ctrl+C` 复制选中的回答文本。

## 8. 主页（主窗口）

从托盘「主页…」或 Dock/任务栏图标打开；平时不常驻（「安静的工具」原则不变，[ADR-19](08-decisions.md)）。**形态：880×560 常规窗口 + 左侧边栏导航**（不是弹窗式信息面板）。

**侧边栏**（180px，`--surface-2` 底）：顶部品牌行（mini 图标 + `Typex`，mini 图标 = 小尺寸 glyph 规则：三柱 + 竖笔）；导航项 `首页 · 历史记录 · 词典`；底部图标行：**设置**（打开独立设置窗口）· **深浅色切换 ◐**（在 亮 ⇄ 暗 间切换，等同修改「设置 → 通用 → 主题」；若当前为「跟随系统」，点击即切为显式主题）。诊断与帮助不在主页放入口——诊断在设置窗口内，帮助走官网/README。

**首页页签**，自上而下：

1. **标题区**：大标题「说，即所得。」+ 快捷键提示行（`按住 [右 ⌘] 开始语音输入 · [右 ⌥] 向助手说指令`，Kbd 组件实时反映用户当前键位）。
2. **统计卡组**（2×2 或 1×4 网格，由本地历史库聚合；关闭历史后停止累计并隐藏）：

| 卡片 | 口径 |
|---|---|
| 总口述时长 | 听写、翻译与助手改写成功注入的录音时长合计；排除助手问答 |
| 口述字数 | 听写、翻译与助手改写最终上屏文本的字数合计；排除助手问答 |
| 节省时间 | 口述字数按打字基准（默认 45 字/分，可在高级设置调整）折算的打字耗时 − 总口述时长，负值显示 0 |
| 平均语速 | 口述字数 ÷ 总口述时长（字/分） |

3. **最近记录**：最新 3 条（时间 · 模式徽标 · 目标应用 · 摘要）+「查看全部历史 →」跳转历史页签；空状态 = 统计归零 + EmptyState（「还没有记录，按住 右⌘ 说第一句话吧」）。

首页不设页脚——版本号、检查更新、数据承诺已在「设置 → 关于」，不重复。

**历史记录页签**（时间倒序）：搜索框 + 「清空全部」；列表行 = 时间 · 模式徽标 · 目标应用 · 文本摘要；点击展开全文。听写/翻译/助手改写显示「原始转写 + 整理后/译文」，助手问答显示「语音指令 + 助手结果」；复制动作始终复制 `result`（助手问答即完整回答），另可删除。列表上方常驻一行小字：「历史仅保存在本机，不含音频」。

**词典页签**：标题 + 说明 + 词条计数（`n / 100`）；输入框 + 添加按钮；搜索框；词条列表。每个词条支持行内编辑、删除，Enter 保存，Esc 取消。当前只显示手动词条，不显示自动/手动来源标记，不支持分类、权重或按应用分组。保存规则：最多 100 条；单条最多 50 个字符；保存时 trim、去空、去重复。词典会进入 STT 阶段和 F-9 整理提示词。

## 9. 错误文案规范

原则：**说人话 + 给出路**。格式 = `哪一步 + 原因 + 动作按钮`。

| 错误分类 | HUD 文案示例 | 动作 |
|---|---|---|
| AuthError | `API 密钥无效（401）` | [打开模型设置] |
| NetworkError | `无法连接 API，请检查网络` | [重试] |
| Timeout | `服务响应超时` | [重试] |
| RateLimited | `请求过于频繁（429），稍后自动重试` | 自动退避重试 |
| ServerError | `服务端错误（5xx）` | [重试] |
| NoSpeech | `没有听到声音` | [选择麦克风] |
| NoFocus | `当前没有输入焦点，结果已复制` | — |
| PermissionMissing | `缺少辅助功能权限，无法输入文字` | [去授权] |
| InjectionBlocked | `目标应用权限更高，结果已复制，请手动粘贴` | — |

所有失败路径必须保住已有产物：STT 成功后任何失败 → `复制原文` 兜底；注入失败 → 自动转剪贴板 + 明示。

## 10. 可访问性

- 全键盘可达；焦点环统一 `--focus-ring`。

## 11. 静态官网

官网是中英双语、亮暗主题的聚焦型单页，不使用路由、账户、Cookie、分析脚本或后端。页面结构固定如下：

1. 窄幅粘性顶栏：Typex 品牌、功能/隐私/下载锚点、GitHub 外链、语言切换、主题切换；窄屏收进菜单，`Escape` 关闭并把焦点还给菜单按钮。
2. 首屏：H1 为 `Typex`，口号为「说，即所得。/ Speak. It types.」，配一句产品定位和 GitHub / 下载两个动作；任意常见桌面与手机首屏都要露出下一节的一部分。
3. 全幅网页演示：通用编辑器场景叠加 Typex HUD，循环展示录音、整理和注入结果。演示仅使用合成电平，不请求麦克风、网络或系统权限；提供播放/暂停控制。
4. 四个交替功能区：任意应用听写与改口清理、即说即译、选区助手、BYOK/本地模型与隐私。每区配一个组件化产品场景，不使用装饰性或冒充原生的截图。
5. 开源与零遥测承诺、macOS / Windows 下载区、GPL-3.0 页脚。

语言首次按浏览器语言选择（`zh*` → 简体中文，其余 → English），手动选择写入 `typex-site-locale`。主题默认跟随 `prefers-color-scheme`，手动切换亮/暗后写入 `typex-site-theme`。两者都必须在首次绘制前应用，避免语言或主题闪烁。

macOS 下载项标注 `macOS 12+`，Windows 标注 `Windows 10 22H2+ / Windows 11 x64`；两个动作都打开 `https://github.com/typex-ink/Typex/releases`，旁边明确说明用户将在 GitHub 选择对应安装包，不伪装成安装包直链。页脚链接到 GPL-3.0 许可证与源码仓库。

官网菜单、按钮、锚点和演示控制全部键盘可达；锚点滚动需避开粘性顶栏。标签页隐藏、演示离屏或用户主动暂停时停止循环；重新可见后仅在用户未暂停且系统未要求 reduced motion 时继续。
