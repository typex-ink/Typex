## 2. 常驻与托盘

- 启动后无任何窗口，仅托盘图标（状态图标规格见 [04 §2.4](04-design-system.md)）。
- 托盘菜单结构：

```
● Typex 就绪                    ← 状态行（不可点，随状态变化：就绪/录音中/处理中/已暂停/错误摘要）
─────────────────
打开助手         （按住 右⌥）
复制上次结果
─────────────────
文本整理           ✓ 开
翻译目标            EN ▸       ← 子菜单：最近语言 + 更多…
模型                  ▸        ← 子菜单：听写/翻译/问答 三组已存档案，✓ 标当前，点选即切（F-4 多档案）
暂停 Typex
─────────────────
设置…               主页…
检查更新             退出
```

- 左键点击托盘 = 打开助手面板；右键 = 菜单（macOS 均为菜单，符合平台惯例）。

## 3. HUD（核心界面）

### 3.1 形态

屏幕**底部居中、距底边 48px** 的全圆角胶囊，高 44px，宽度随状态 160→320px 平滑过渡；毛玻璃背景（`--surface` 半透明 + blur，Linux 回退实心）；**永不获得焦点**。多显示器时出现在「当前焦点窗口所在屏」。

### 3.2 状态与布局

**录音中（听写）**：

```
╭──────────────────────────────────╮
│ ● 0:07   ▁▃▅▇▅▃▁▂▄▆▄▂▁   [听写] ✕│
╰──────────────────────────────────╯
   │        │                │    └ 取消（Esc）
   │        │                └ 模式徽标（听写/中→EN/助手）
   │        └ 实时波形（--voice 色，Canvas）
   └ 录音点（--recording，1s 呼吸）+ 计时
```

- 翻译模式：徽标显示 `中 → EN`（点击弹出目标语言快切）；助手模式：徽标 `助手`。
- 波形必须真实反映电平——用户要能一眼确认「麦克风在收音」。收不到声音 3 秒 → 波形区显示「没有听到声音」微提示。

**处理中**：波形区渐变为五柱依次呼吸动画 + 文案（`正在转写…` → `正在整理…`/`正在翻译…`）；超过 5 s 显示已耗时。

**成功**：胶囊短促回弹，显示 `✓ 已输入`（`--success`），600 ms 后下滑淡出。

**失败**：

```
╭──────────────────────────────────────────╮
│ ⚠ 转写失败：无法连接 API    [重试] [复制原文] ✕ │
╰──────────────────────────────────────────╯
```

- 错误文案 = 错误分类的人话（见 §9）；`复制原文` 仅在 STT 已成功、后续步骤失败时出现（不丢话铁律）。失败态不自动消失。

**原样模式提示**：整理层关闭时，录音态徽标处加小字 `原样`。

### 3.3 状态机与时序

```
hidden ─按键按下─▶ recording ─松开─▶ processing ─▶ success(600ms) ─▶ hidden
   ▲                  │ Esc/✕            │ 失败                        
   └──────────────────┴◀────────────── failed ─重试─▶ processing
                                          └─✕/超时120s─▶ hidden
```

出现动画 220 ms spring 上滑；按下到 HUD 可见 ≤ 100 ms（性能预算）。

**重入规则**：Transcribing/Processing/Injecting 期间再次按下触发键 → 本次按键忽略，HUD 轻微晃动 + 微文案「正在处理上一条…」（v1 不做流水线并行，简单可预期；失败态下按触发键 = 放弃上一条并开始新录音）。

**Esc 取消的边界**：Esc 仅在 recording 状态被监听，且为 listen-only **不吞键**（按键仍传给前台应用，避免干扰 vim 等重度 Esc 用户之外还吞掉他们的按键）；副作用可接受（取消录音本来就是用户意图的大概率解释）。设置中可关闭「Esc 取消录音」。

## 4. 助手面板（F-3）

### 4.1 形态

屏幕上 1/3 居中，宽 560px，圆角 16，毛玻璃 + `--shadow`；可获得焦点；失焦自动隐藏（右上角 📌 可固定）。

### 4.2 布局

```
╭────────────────────────────────────────────────╮
│ ⌗ 来自 Chrome · 217 字                      ✕  │  ← 上下文芯片（有选中文本时；✕移除）
│                                                │
│  🎤 按住说话，或输入问题…                 [🎙] │  ← 输入行（语音键按住=录音）
├────────────────────────────────────────────────┤
│  （流式回答区域，Markdown 渲染，代码块可复制）    │
│                                                │
│  ┌──────────────────────────────────────────┐  │
│  │ Sure — here's a more concise version: …  │  │
│  └──────────────────────────────────────────┘  │
│                                                │
│  [替换选区 ⏎]  [插入到光标]  [复制]        📌  │  ← 动作行（无选区时无「替换」）
╰────────────────────────────────────────────────╯
```

### 4.3 行为

- **单轮语义**：新提问清空上一条回答（面板保留最近一条以便对照，被新回答覆盖）；无会话列表、无历史线程。
- F-3a 自动替换开启时：改写型结果收完 → 直接替换选区 → 面板显示 `✓ 已替换` 并 800 ms 后收起；回答型结果（`ANSWER:` 信号，见 [03 §3.4](03-model-providers.md)）始终留在面板。
- 键盘：`⏎` 执行主动作（有选区=替换，无=复制）；`Esc` 关闭；`⌘/Ctrl+C` 复制回答。
- 流式渲染节流 30 fps；渲染 Markdown（标题/列表/代码块），链接点击用系统浏览器打开。

## 5. 设置窗口

左侧图标+文字导航（宽 160px），右侧内容区。导航：`通用 · 听写 · 翻译 · 助手 · 模型服务 · 快捷键 · 历史 · 诊断 · 关于`。

### 5.1 「模型服务」页（最重要的一页）

```
┌─────────────┬──────────────────────────────────────────┐
│ 通用         │  模型服务                                 │
│ 听写         │                                          │
│ 翻译         │  ── 语音转文字 ────────────────────────   │
│ 助手         │  ┌ ProviderCard ─────────────────────┐   │
│ ▶模型服务    │  │ [logo] Groq · whisper-v3-turbo    │   │
│ 快捷键       │  │ openai_compat · 已连接 · 320ms     │   │
│ 历史         │  │            [测试] [编辑] [切换 ▾] │   │
│ 诊断         │  └───────────────────────────────────┘   │
│ 关于         │                                          │
│             │  ── 文本整理 ──────────── [与翻译共用 ✓]  │
│             │  ── 翻译模型 ────────────────────────     │
│             │  ── 问答模型 ────────────────────────     │
└─────────────┴──────────────────────────────────────────┘
```

- 每个槽位一张 `ProviderCard`；「编辑」进入**编辑子页**（同一内容区内切换，顶部「← 模型服务」返回链接；保存/删除档案后自动返回列表；有未保存修改时离开需确认）：预设下拉（OpenAI/Groq/SiliconFlow/火山·豆包/DeepSeek/OpenRouter/Ollama/自定义，仅作表单填充、无推荐标注）→ 按 `kind` 动态渲染字段（openai_compat：Base URL/模型/密钥；volcengine：AppKey/AccessToken/ResourceId）。「已下载模型」管理页同为子页，同样以「← 模型服务」返回。
- **「切换 ▾」= 多配置档案（[ADR-21](09-decisions.md)）**：下拉列出该槽位所有已存档案（✓ 标当前）+「新建配置…」；新建不覆盖旧档案，切换即时生效。同一组档案也出现在托盘「模型 ▸」子菜单中（快速切换，不用打开设置）。
- 密钥输入用 `SecretInput`（默认掩码、可见性切换、粘贴按钮）；保存即写 keyring。
- 「测试」按钮内联显示结果：`✓ 320ms` 或分类错误文案 + 修复建议。
- **本地 Provider 卡片**（v1.1）：预设列表含「本地 · 离线」；卡片副标题显示引擎与模型（如 `local · Qwen3.5-2B`）及模型状态（`已下载 · 1.3 GB` / `未下载 [下载]`，下载中显示进度条）；无密钥字段，编辑态字段 = 模型（下拉，来自模型库）+ 加载策略（常驻 / 录音时预热）；「测试」= 本地跑一次内置样音/ping。问答槽的预设列表**不出现**本地选项（[ADR-20](09-decisions.md)）。
- **已下载模型管理**（页面底部「管理…」进入）：已下载列表（名称 · 用途 · 体积 · 被哪些槽使用 · 删除，删除使用中模型先警告）+ 模型库可下载列表（行内显示硬件要求与本机检测结果，如「需 GPU 加速（本机 ✓ Metal）」，[ADR-22](09-decisions.md)）+ 占用合计与下载源切换。
- LLM 槽位卡片右上有「与 X 共用连接」开关（见 [03 §5](03-model-providers.md) 共用规则）。

### 5.2 其他页要点

- **通用**：语言（跟随系统/中/英）、主题（跟随系统/亮/暗）、开机自启、提示音开关与音量、网络代理（跟随系统/手动/直连）、更新通道。
- **听写**：文本整理开关、整理提示词（高级，折叠；展开为模板编辑器——占位符 `{transcript}`（原始转写，必需）/`{dictionary}`（个人词典，可选）高亮显示，缺必需占位符时禁用保存并行内报错，「恢复默认」回内置模板；占位符规格见 [03 §3.4](03-model-providers.md)）、注入方式（自动/剪贴板粘贴/逐字输入）、粘贴延迟滑杆、麦克风设备选择 + 电平实时预览条。
- **翻译**：源语言（你说话使用的语言）、目标语言（要翻译成的语言）、双向翻译开关（检测到说的是目标语言时自动译回源语言）、翻译提示词（高级，折叠，置底；模板编辑器同听写，占位符 `{transcript}`/`{source_language}`/`{target_language}`）。
- **助手**：结果处置（自动替换/预览确认）、问答提示词（高级，折叠；模板编辑器同听写，占位符 `{instruction}`（语音指令，必需）/`{selection}`（选中文本，可选））。
- **诊断**：环境自检（平台/compositor/可用注入后端/权限状态，每项 ✓/✗ + 修复引导按钮）、日志目录、导出诊断包（自动脱敏）。
- **历史**：开关（默认开，附一句说明：仅存本机、不含音频）、保留期限（默认 90 天）、清空按钮。

## 7. 快捷键

### 7.1 默认键位（已定，[ADR-7](09-decisions.md)）：全修饰键三角方案

| 功能 | macOS | Windows / Linux | 行为 |
|---|---|---|---|
| 听写 | 按住 `右 ⌘` | 按住 `右 Ctrl` | 长按=说话，短按=切换录音 |
| 助手 | 按住 `右 ⌥` | 按住 `右 Alt` | 按住=呼出面板并录音；短按=仅呼出面板 |
| 翻译 | 按住 `右 ⌘ + 右 ⌥` | 按住 `右 Ctrl + 右 Alt` | 两键按住（不分先后）=翻译录音 |

设计逻辑：**只用修饰键，不碰任何普通键**。修饰键「单独按住」在几乎所有软件里是空操作，右侧修饰键又极少参与日常组合键——这是冲突概率最低的一类键位；三键有记忆结构（A=输入，B=问答，A+B=输入的变体：翻译）。

**乐观启动与模式判定**：任一触发键按下即开始录音；350 ms 判定窗口内组合出另一个触发键 → 无缝切换为翻译模式（音频保留、HUD 徽标即时变化）；录音中追加按下另一键同样即时升级为翻译。

**配套规则（实现必须遵守，见 [07 §7.3](07-code-architecture.md)）**：

1. **组合键让路**：按住触发键期间若按下任何**普通键**（用户实际在用 `⌘C`、`AltGr+E` 等系统/输入组合键），立即静默取消本次录音并完全放行按键——右⌘/右Alt 的日常用法零干扰。这条规则是单修饰键触发可行性的根基。
2. **Windows Alt 单击吞键**：单击 Alt 会聚焦应用菜单栏，助手键短按时须经低级键盘钩子吞掉该次 keyup，防止菜单栏被误激活（仅此一处需要事件拦截，比拦截 `Ctrl+Space` 类组合简单得多）。
3. AltGr 布局（欧洲键盘）：右 Alt 物理键即 AltGr——规则 1 已保证 `AltGr+字母` 打特殊字符不受影响；单独按住 AltGr 说话无副作用。

**已知残余情况**（onboarding 第 4 步当场改键兜底）：MacBook 内置键盘无右 Ctrl（故 macOS 用右 ⌘）；用户用 Karabiner/PowerToys 重映射过右侧修饰键；HHKB 等无右修饰键的小配列键盘。`HotkeyRecorder` 支持单修饰键、修饰键组合与普通键，录制时对系统热键表与常见软件冲突黄条警告；另提供预设建议：CapsLock（提示 macOS 中文用户注意中英切换冲突）、F13–F19、右键菜单键（仅 Win/Linux）。

### 7.2 面板内快捷键

`Esc` 关闭/取消 · `⏎` 主动作 · `⌘/Ctrl+,` 打开设置 · 按住 `🎙` 或全局助手键在面板内继续语音输入。

## 8. 主页（主窗口，P1）

从托盘「主页…」或 Dock/任务栏图标打开；平时不常驻（「安静的工具」原则不变，[ADR-19](09-decisions.md)）。**形态：880×560 常规窗口 + 左侧边栏导航**（不是弹窗式信息面板）。

**侧边栏**（180px，`--surface-2` 底）：顶部品牌行（mini 图标 + `Typex`，mini 图标 = 小尺寸 glyph 规则：三柱 + 竖笔）；导航项 `首页 · 历史记录`（F-10 个人词典上线时增加「词汇表」项）；底部图标行：**设置**（打开独立设置窗口）· **深浅色切换 ◐**（在 亮 ⇄ 暗 间切换，等同修改「设置 → 通用 → 主题」；若当前为「跟随系统」，点击即切为显式主题）。诊断与帮助不在主页放入口——诊断在设置窗口内，帮助走官网/README。

**首页页签**，自上而下：

1. **标题区**：大标题「说，即所得。」+ 快捷键提示行（`按住 [右 ⌘] 开始语音输入 · [右 ⌥] 呼出助手`，Kbd 组件实时反映用户当前键位）。
2. **统计卡组**（2×2 或 1×4 网格，由本地历史库聚合；关闭历史后停止累计并隐藏）：

| 卡片 | 口径 |
|---|---|
| 总口述时长 | 全部会话的录音时长合计 |
| 口述字数 | 全部会话最终上屏文本的字数合计 |
| 节省时间 | 口述字数按打字基准（默认 45 字/分，可在高级设置调整）折算的打字耗时 − 总口述时长，负值显示 0 |
| 平均语速 | 口述字数 ÷ 总口述时长（字/分） |

3. **最近记录**：最新 3 条（时间 · 模式徽标 · 目标应用 · 摘要）+「查看全部历史 →」跳转历史页签；空状态 = 统计归零 + EmptyState（「还没有记录，按住 右⌘ 说第一句话吧」）。

首页不设页脚——版本号、检查更新、数据承诺已在「设置 → 关于」，不重复。

**历史记录页签**（时间倒序）：搜索框 + 「清空全部」；列表行 = 时间 · 模式徽标 · 目标应用 · 文本摘要；点击展开全文（原始转写 + 整理后/译文 双栏对照），行内动作：复制 / 删除。列表上方常驻一行小字：「历史仅保存在本机，不含音频」。

## 9. 错误文案规范

原则：**说人话 + 给出路**。格式 = `哪一步 + 原因 + 动作按钮`。

| 错误分类 | HUD 文案示例 | 动作 |
|---|---|---|
| AuthError | `API 密钥无效（401）` | [打开模型设置] |
| NetworkError | `无法连接 API，请检查网络` | [重试] |
| Timeout | `服务响应超时` | [重试] |
| RateLimited | `请求过于频繁（429），稍后自动重试` | 自动退避重试 |
| ServerError | `服务端错误（5xx）` | [重试] |
| NoSpeech | `没有听到声音` | [选择麦克风] |
| NoFocus | `当前没有输入焦点，结果已复制` | — |
| PermissionMissing | `缺少辅助功能权限，无法输入文字` | [去授权] |

所有失败路径必须保住已有产物：STT 成功后任何失败 → `复制原文` 兜底；注入失败 → 自动转剪贴板 + 明示。

## 10. 可访问性

- 全键盘可达；焦点环统一 `--focus-ring`。
- HUD 状态变化经系统无障碍通告（screen reader announce）。
- 尊重 `prefers-reduced-motion`（见 05 §6）与系统字号缩放（rem 布局）。
- 色彩不作为唯一状态信号（图标/文字并行）。
