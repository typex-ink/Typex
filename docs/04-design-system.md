# 04 · 设计系统（品牌 · 图标 · 配色 · 字体 · 动效）

> Typex 产品设计书 · 第四章
> 本章 token 是前端 `src/lib/tokens.css` 的唯一事实来源；界面应用见 [05 UX 规格](05-ux-spec.md)。
> **可视化配套物见 [`mockups/`](mockups/README.md)**：定稿 HTML mockup、渲染截图、tokens.css 起步文件——用 AI 辅助实现 UI 时把它们一并提供给模型，避免模型凭空想象界面外观。

## 1. 品牌人格

**克制、迅捷、聪明。** Typex 是一件安静的工具，不是一个吵闹的 AI 产品。视觉语言是**纯墨（Ink）**：整个产品只用黑、白、灰三色构成——域名 typex.ink 的字面执行，「墨落在纸上，声音变成文字」。层级不靠色相，靠明度与结构（实底 / 灰底 / 发丝线 / 下划线 / 字重）。

唯一保留的彩色是**录音红**——它不是装饰，是隐私功能色：用户必须零歧义地知道麦克风开着（与 macOS/iOS 状态栏录音指示同一语义）。错误红与之同族。除此之外界面上不出现任何色相。

用户 90% 的时间只看到一枚托盘图标和一个小小的 HUD 胶囊，所以这两处要做到像素级精致。

关键词：`quiet` · `ink` · `precise`。隐喻：**声波穿过字母，变成文字**。

## 2. 图标设计

### 2.1 主图标概念：「声波成 T」（Waveform T）

字母 **T** 的横梁由 5 根竖直的**波形柱**构成（中间高、两侧低，模拟说话时的电平），竖笔是一根**文本插入光标（I-beam 的简化竖线）**——一眼读出「声音（波形）落下来变成文字（光标）」，同时就是 Typex 的首字母。

```
     ▂ ▄ █ ▄ ▂        ← 5 根波形柱构成 T 的横梁
         │
         │            ← 竖笔 = 文本光标
```

结构规范（以 1024×1024 网格为基准）：

- 波形柱宽 79，柱间距 59，五柱高度比 `0.40 : 0.70 : 1 : 0.70 : 0.40`（最高柱 394）；柱两端全圆角（半径 = 柱宽/2）。
- 竖笔宽 79，高 276，顶端与中柱底部留 197 间隙（**不相连**——留出的负空间是「声音落到光标上」的瞬间感）；不加底衬。
- 图形源自应用内 mini glyph，整体在安全区内居中。Dock/app 交付图标在 1024 画布内保留 72px 透明安全边，让白色 squircle 在 Dock 中不显得过大。

### 2.2 App 图标（各平台交付物）

图标只用**纯黑 `#000000` 与纯白 `#FFFFFF`** 两色，不含灰阶、不含渐变：

- **纸版（Dock / app 默认）**：纯白底 + 纯黑 glyph。
- **墨版（界面浅色模式内联变体）**：纯黑底 + 纯白 glyph（五柱与竖笔同为白）。

配对规则与界面相反相成：**浅色环境用墨版（黑底白字），深色环境用纸版（白底黑字）**——图标永远是环境里对比最强的一枚「印章」。图标的记忆点由 glyph 轮廓本身（波形 + 光标的负空间）承担，不依赖颜色。

| 平台 | 形状与背景 | 交付尺寸 |
|---|---|---|
| macOS | 系统 squircle 蒙版；默认交付纸版（Dock 图标为静态资源，不随主题自动切换，优先展示白底高对比图标）；墨版仅作为界面内联变体 | `.icns`（16–1024 全套，@1x/@2x） |
| Windows | 同一设计直接用圆角矩形（半径 27%），默认交付纸版 | `.ico`（16/24/32/48/64/256） |
| Linux | 同 Windows 设计 | hicolor PNG 16–512 + SVG |

小尺寸（≤32px）仍保留五柱 + 竖笔，与应用内 mini 图标一致。

源文件：`assets/icon/typex.svg`（单一 SVG 源，脚本导出全平台，禁止手改位图）。

### 2.3 Logo 与文字标

- 横排 logotype：glyph + `Typex` 字样（Inter SemiBold，字距 -1%）。**五个字母永远同色、不含灰色成分**：浅色模式全黑 `#000000`，深色模式全白 `#FFFFFF`。用于 README、官网、关于页。
- 不存在渐变或多色版本；单色即品牌。

### 2.4 托盘 / 菜单栏图标（状态系统）

托盘图标 = 去掉竖笔的**五柱波形 glyph**（小尺寸下比完整 T 清晰）。macOS 用 template image（系统自动适配深浅/高亮），Windows/Linux 提供深浅两套。

| 状态 | 表现 |
|---|---|
| 空闲 | 静态五柱 |
| 录音中 | 五柱替换为**实时电平动画**（8 fps 足够，省电）；Windows/Linux 加**录音红**小圆点角标（`--recording`） |
| 处理中 | 柱体从左到右依次呼吸（透明度 40%↔100% 循环） |
| 暂停 | 五柱降为 40% 透明度 + 斜杠角标 |
| 错误 | 静态五柱 + 红色角标圆点（`--error`） |

## 3. 配色方案（纯墨 Monochrome）

### 3.1 设计逻辑

- **黑白灰三色构成整个界面**，层级全靠明度与结构表达，不靠色相：主动作 = 墨色实底；选中 = 灰底 + 字重（+ ✓ 标记）；链接 = 下划线；边界 = 发丝线。
- 中性灰带**极轻微暖调**（不用纯灰，避免「工程工具感」）。
- **唯一的彩色是红**：`--recording` 用于「正在录音」指示（隐私功能色，用户必须零歧义地知道麦克风开着，非装饰、不可灰化），`--error` 用于错误。两者同族不同值。
- 成功/警告**灰阶化**，语义由符号（✓ / ⚠）+ 明度承担，不再占用绿/黄色相。
- 波形与电平使用墨色（`--voice` 保留为语义 token，当前值 = `--text-1`；录音动画本身即「活着」的信号）。
- 主按钮是页面上唯一保持实底反色的元素（一个页面需要一处最重的锚点）；侧边栏选中、卡片选中、tag 一律用 `--sel-bg` 灰底，**禁止反色实底**（刺眼）与粗黑描边。
- 所有正文级前景/背景组合满足 WCAG AA（≥ 4.5:1），已逐项校验。

### 3.3 渐变与彩色的禁令

品牌不使用任何渐变（原 Aurora 渐变已废弃）。除 `--recording` / `--recording-soft` / `--error` 外，任何界面元素不得引入色相；插画、onboarding、空状态一律以灰阶 + 线条表达。若未来需要「点亮」某处，用**明度对比**（纯白/纯黑）而不是颜色。

### 3.4 主题切换

默认**跟随系统**（`prefers-color-scheme`），可手动固定亮/暗。所有颜色一律经 CSS 变量引用，组件内禁止硬编码色值（lint 规则保障）。HUD 与托盘同样双主题。

## 4. 字体

| 用途 | 字体栈 |
|---|---|
| UI 文本 | `Inter, "SF Pro Text", "Segoe UI Variable", system-ui, "PingFang SC", "Microsoft YaHei UI", "Noto Sans CJK SC", sans-serif`（Inter 4.x 可变字体随包内置，≈300 KB woff2） |
| 等宽（端点/密钥/日志） | `"JetBrains Mono", "SF Mono", Consolas, "Noto Sans Mono CJK SC", monospace`（不随包内置，系统栈优先） |

字号阶梯（桌面密度）：`11 caption · 12 secondary · 13 body（默认）· 15 title · 20 heading · 28 display`；行高 1.5（正文）/ 1.2（标题）；中文场景字重上限 600（避免 CJK 渲染发糊）。

## 5. 形状、间距、层级

- 间距基数 4px；常用 8/12/16/24。
- 圆角：控件 8 · 卡片/面板 12 · 独立浮窗 16 · **HUD 胶囊 = 全圆角**。
- 层级靠「表面色阶 + 发丝线 + 阴影」表达，禁止用不透明度叠黑白。
- 毛玻璃：HUD 与助手面板 `backdrop-filter: blur(20px)` + 半透明 surface（macOS/Windows 可用；Linux/webkit2gtk 探测失败自动回退实心 `--surface`——**回退必须同样好看**，用 98% 不透明度）。

## 6. 动效

| 场景 | 规格 |
|---|---|
| HUD 出现/消失 | 220 ms，spring（stiffness 380 / damping 30）：自屏幕底边缘上滑 12px + 淡入；消失 160 ms 淡出下滑 |
| 波形 | Canvas 60 fps，柱高对电平做 80 ms 平滑；**只在录音时运行**（空闲 CPU≈0 铁律） |
| 状态切换（录音→处理→成功） | 200 ms 交叉淡入；成功时胶囊短促「回弹」1 次（scale 1→1.04→1） |
| 处理中 | 五柱依次呼吸 + 高光微光（前景色 12% 不透明度）从左到右 1.6 s 循环 |
| 按钮/hover | 120 ms ease-out |
| 全局 | 尊重 `prefers-reduced-motion`：动画替换为 80 ms 淡入淡出；波形降为静态电平条 |

提示音（`assets/sounds/`，皆 < 150 ms、-14 LUFS、可关闭）：开始 = 上行双音「叮·咚」（清爽、不刺耳）；成功 = 单音短「嗒」；错误 = 低频闷「噗」。三枚音色同一乐器族（木琴系），构成品牌听觉签名。

## 7. 组件规范（前端 `src/lib/components/`）

`Button`（primary/secondary/ghost/danger，高 32，radius 8）· `Input`/`SecretInput`（密钥用，带显隐与粘贴按钮，高 32）· `Select`（应用内自绘 listbox，禁止使用系统原生下拉；支持方向键/Enter/Esc、外部点击关闭）· `Toggle` · `HotkeyRecorder`（录制快捷键专用控件，见 06 §7）· `ProviderCard`（配置档案卡片：厂商 logo 位 + label + 模型名 + 测试按钮 + 延迟徽标）· `Waveform`（Canvas 封装，输入电平数组）· `StatusPill`（HUD 核心容器）· `Kbd`（快捷键展示）· `EmptyState` · `Callout`（info/warn/error）。

所有组件必须：双主题可用、键盘可达（Tab/Enter/Esc）、焦点环使用 `--focus-ring`、文案取自 i18n 资源（中英双语，[ADR-11](09-decisions.md)）。
