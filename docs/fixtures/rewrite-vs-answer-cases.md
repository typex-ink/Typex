# 改写 vs 回答判定样例集（F-3a）

> 处理提示词判定行为的回归基准（08 §6）。对应 F-3 验收标准 1/2：
> 改写型指令 → 直接输出结果（无 `ANSWER:` 前缀，替换选区）；提问型 → `ANSWER:` 前缀（弹窗展示，不动选区）。
> 每例含选中文本 + 语音指令与**期望判定**；执行：`scripts/eval-prompts.ts`（需环境变量密钥）。
> 判定铁律：宁可误判为回答（不替换），不可误判为改写（误替换）。

## A. 改写型（期望：无 `ANSWER:` 前缀，输出可直接替换选区，≥10）

| # | 选中文本 | 语音指令 | 期望要点 |
|---|---|---|---|
| A1 | 这个方案我觉得还行吧，应该问题不大 | 改得正式一点 | 无前缀；无解释寒暄；语义保留「方案可行」 |
| A2 | The meeting is at 3pm tomorrow. | 翻译成中文 | 无前缀；含「明天」「三点」（或「3 点」）；无英文原句残留 |
| A3 | 我们计划在下个季度推出三个新功能，分别是深色模式、离线支持和多语言界面，同时还会优化性能 | 精简一半 | 无前缀；字数明显少于原文；保留三个功能点 |
| A4 | {"name":"typex","version":"1.0","license":"GPLv3"} | 把这段 JSON 格式化 | 无前缀；输出为多行缩进 JSON；键值不变 |
| A5 | 你好张总我想约下周三下午聊一下合作的事情谢谢 | 加上标点断句 | 无前缀；含标点；内容不增删 |
| A6 | i cant beleive this actualy works | 修正拼写和大小写 | 无前缀；含 "can't"（或 "cannot"）、"believe"、"actually" |
| A7 | 苹果、香蕉、橙子、葡萄 | 改成竖排清单 | 无前缀；四行列表（- 或序号） |
| A8 | 这个 bug 是因为空指针引起的 | 翻译成日语 | 无前缀；输出为日语 |
| A9 | 亲爱的客户您好，感谢您选择我们的产品…… | 改成一句话摘要 | 无前缀；一句话；无解释 |
| A10 | fn main() { println!("hi") } | 加上注释 | 无前缀；保留原代码；含注释符号 |

## B. 提问型（期望：`ANSWER:` 前缀，≥10）

| # | 选中文本 | 语音指令 | 期望要点 |
|---|---|---|---|
| B1 | TypeError: Cannot read properties of undefined (reading 'map') | 这是什么意思 | 有前缀；解释 undefined 上调用 map |
| B2 | 曲径通幽处，禅房花木深 | 这是谁写的 | 有前缀；含「常建」 |
| B3 | SELECT * FROM users WHERE id = 1 | 这句 SQL 有性能问题吗 | 有前缀；是回答而非改写 SQL |
| B4 | E: Unable to locate package libssl-dev | 怎么解决 | 有前缀；含解决步骤（apt update 等） |
| B5 | 本合同自双方签字之日起生效，有效期三年 | 有效期到什么时候算 | 有前缀；回答基于「签字日 + 三年」 |
| B6 | GPLv3 | 这个许可证允许商用吗 | 有前缀；回答允许/条件 |
| B7 | 内存占用 144.5 MB | 这个数字算高吗 | 有前缀；是评价性回答 |
| B8 | panic: runtime error: index out of range [3] with length 3 | 为什么会越界 | 有前缀；解释下标从 0 开始 |
| B9 | 每天一杯咖啡大约含 95 毫克咖啡因 | 一天喝几杯安全 | 有前缀；含数量建议 |
| B10 | let x = 5; x = 6; | 这段 Rust 代码能编译吗 | 有前缀；回答不能（缺 mut） |

## C. 边界样例（易误判，判定必须稳定）

| # | 选中文本 | 语音指令 | 期望判定 | 说明 |
|---|---|---|---|---|
| C1 | 这段话写得怎么样 | 翻译成英文 | 改写 | 选中文本本身是问句，但指令是加工 |
| C2 | Hello world | 这是什么语言写的 | 回答 | 指令含「什么」提问词 |
| C3 | 把大象放进冰箱需要几步 | 改得更口语化 | 改写 | 选区是问句 + 指令是改写 |
| C4 | 3.1415926 | 帮我翻译一下 | 改写（或回答均可接受） | 数字无从翻译——模型可原样输出或说明，不得编造 |
| C5 | rm -rf / | 这个命令危险吗 | 回答 | 危险命令解释，绝不能「执行式改写」 |
